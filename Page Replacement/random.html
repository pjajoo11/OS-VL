Random Page Replacement Algorithm

  • Random replacement algorithm replaces a random page in memory.

  • This eliminates the overhead cost of tracking page references.

  • In page replacement using a random algorithm (random) every time a page fault occurs, the replaced page is randomly selected. This technique does not use any information in determining which pages to replace, all pages in main memory have the same weight to be selected.

  • How to select random or arbitrary pages, including the page that is currently referenced (pages that should not be replaced).

  • This random technique is very bad, the experiment shows the random algorithm causes a very high frequency of page faults.

  • Usually it fares better than FIFO, and for looping memory references it is better than LRU, although generally LRU performs better in practice.

  • OS/390 uses global LRU approximation and falls back to random replacement when LRU performance degenerates, and the Intel i860 processor used a random replacement policy
